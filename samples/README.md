# Samples Guide

Samples are focused, user-facing demonstrations of one engine capability at a time.

│  ├─ input/

│  ├─ physics/
│  ├─ tilemap/
│  ├─ audio/
│  ├─ ui/
│  ├─ etc.../

## Rules
- One sample = one capability
- Keep minimal and clear
- No games, no tests
