Plan Notes Before Pong

  1. Keep Pong as a focused PR with one implementation pass.
  2. Build arcade-faithful core play first: serve, rally, score, and win/reset loop.
  3. Add paddle-hit english and gamepad support in the first delivery, not as later cleanup.
  4. Structure rules so Tennis, Hockey, Handball, and Jai Alai can slot in as variants instead of separate games.
  5. Use the tools hub to guide asset creation, palette workflow, and future editing tools.
Roadmap entries are planning targets only; they are intentionally shown without live links in this PR.
