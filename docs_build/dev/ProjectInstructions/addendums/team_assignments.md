# Team Assignment Governance

## Assignment Source

Teams pull work from `docs_build/dev/ProjectInstructions/backlog/BACKLOG_MASTER.md`.

## Assignment Flow

1. Select one backlog item.
2. Change the backlog status to `[.]` Building.
3. Add the item under the owning team in `TEAM_ASSIGNMENTS.md`.
4. Create or continue the team's active branch.
5. Work only on that assignment.

## Limits

- One active assignment per team.
- One active branch per team unless a MASTER override approves otherwise.
- A team must not work outside its active assignment.

## MASTER Override

The required override wording is:

`MASTER override approved: <reason>`

The reason must identify the assignment, branch, scope, or ownership exception being approved.
