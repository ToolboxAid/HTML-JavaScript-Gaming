# PR_11_194 — V2 HTML-First Remaining Tools Batch

## Purpose
Continue the V2 re-engineer lane with a larger, testable Codex-driven PR. Codex must move remaining V2 tool static shell/layout/header/CSS into each tool's `index.html` and keep each `index.js` behavior-only.

## Scope
- V2 tools only.
- No schema changes.
- No sample changes.
- No game changes.
- No Workspace Manager v1 work.
- No platformShell usage.
- No `toolbox/shared/*` usage.
- No legacy copy/paste.

## Required Architecture
Each V2 tool must use:

```text
toolbox/<tool>-v2/index.html  -> static shell, CSS links, shared header mount, existing accordion markup, app containers
toolbox/<tool>-v2/index.js    -> single class, session read, validation, DOM population, render, empty/error states
```

## Required Header
Each V2 tool HTML shell must include:

```html
<div id="shared-theme-header"></div>
```

The shared theme header must be mounted through the same mechanism used by the repo index/home pages. Do not create a custom header system.

## Required Naming
All new V2 tool visible names and page titles must end with `V2`.

Examples:
- Palette Manager V2
- SVG Asset Studio V2
- Vector Map Editor V2
- Tilemap Studio V2
- Asset Browser V2

## Behavior Rules
- Session-backed data only.
- Tool direct reads session via `hostContextId`.
- Tool URL writes session then tool reads session.
- Workspace writes session then tool reads session.
- Tool never fetches, guesses, or uses fallback/default data.

## Codex Must Remove From V2 JS
- `document.body.innerHTML = ...`
- `document.head.insertAdjacentHTML(...)`
- dynamic style injection
- dynamic shared-header script injection
- layout construction in JS
- header construction in JS
- alias variables
- helper classes
- abstraction layers

## Validation
Codex must run targeted checks only:

```powershell
node --check toolbox/<affected-tool-v2>/index.js
```

Codex must also document manual browser validation steps for each affected V2 tool.

Do not run full samples smoke test unless Codex modifies shared sample loader/framework, which this PR must not do.

## Evidence Required
Codex must write a report under:

```text
docs_build/dev/reports/PR_11_194_v2_html_first_remaining_tools_report.md
```

The report must include:
- files changed
- V2 tools affected
- confirmation no schema/sample/game/workspace-v1 files changed
- validation commands run
- whether full samples smoke test was skipped and why
- final Codex output ZIP path

## Done When
- Remaining V2 tools use HTML-first shell.
- V2 JS files are behavior-only.
- Shared header mount is present in HTML.
- V2 suffix naming is enforced.
- Session-only rules remain intact.
- Targeted validation passes.
