# Public Platform Showcase Video Plan

## Target Length
- 30 to 60 seconds

## Sequence
1. Title card
   - Show platform name and vector-native positioning.
   - Overlay: `Verified vector-native platform`.
2. Flagship title screen
   - Show Asteroids title state.
   - Overlay: `Flagship demo runs on the accepted platform path`.
3. Gameplay loop
   - Show ship movement, firing, asteroid splits, score, lives, and wave progression.
   - Overlay: `Vector-only runtime with strict validation and packaging`.
4. Game-over and restart
   - Show loop continuity.
   - Overlay: `Title -> gameplay -> game over -> restart`.
5. Debug surface
   - Show dependency graph and runtime trace.
   - Overlay: `Debug view stays aligned to packaged/runtime state`.
6. Profiler surface
   - Show `geometry`, `packaging`, and `runtime` profiler lines.
   - Overlay: `Deterministic geometry runtime is visible and measurable`.
7. Template proof
   - Show the standalone vector-native template sample.
   - Overlay: `Template reuse proven beyond the flagship demo`.
8. Close card
   - Overlay:
     - `Validation ready`
     - `Packaging ready`
     - `Runtime ready`
     - `Export ready`
     - `Publishing ready`
     - `110/110 node tests passed`

## Presentation Rules
- Keep overlays short and factual.
- Do not add claims not backed by the verified repo outputs.
- Keep the capture silent-friendly and readable without narration.
