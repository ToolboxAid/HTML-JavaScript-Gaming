Below is a **full, production-grade Game Design Document (GDD)** for reverse-engineering *Joust* (Williams, 1982), including **flap-based flight physics, enemy AI tiers, egg/hatch systems, lava hazard logic, wave progression, and finishing touches** for an arcade-accurate recreation.

# 🎮 GDD — Joust (Reverse Engineering Edition)

**Document Type:** Full Technical + Design Spec
**Target:** Faithful arcade recreation (Williams feel, physics-driven)
**Genre:** Action / Arena Platformer
**Perspective:** 2D Side View (single-screen arena)
**Core Identity:** Flap-based flight + altitude combat + timing mastery

---

# 1. 🧭 CORE GAME OVERVIEW

## 1.1 Game Pillars

* **Flap-Based Flight Control (NOT jumping)**
* **Altitude-Based Combat Priority**
* **Egg Lifecycle (kill → egg → hatch → escalation)**
* **Arena Awareness + Wraparound**
* **Escalating Chaos**

---

## 1.2 Player Fantasy

You are a knight riding a flying ostrich, defeating enemy riders by striking from above while managing the battlefield.

---

# 2. 🎮 CORE GAMEPLAY LOOP

1. Spawn in arena
2. Flap to gain altitude
3. Engage enemies
4. Defeat enemies → eggs drop
5. Collect eggs OR allow them to hatch
6. Survive hazards (lava, pterodactyl)
7. Clear wave → next wave (harder)
8. Death → respawn (if lives remain)

---

# 3. 🪶 PLAYER SYSTEM

## 3.1 Controls

* Left / Right movement
* Flap (single button)

---

## 3.2 Movement Model (CRITICAL)

### Flap-Based Physics

```text id="s8u6yy"
velocityY += flapForce
velocityY -= gravity
```

---

### Properties

```text id="2kw8zv"
Flap Force: Fixed impulse
Gravity: Constant downward
Horizontal Speed: Moderate
Inertia: Present but light
```

---

## 3.3 Flight Behavior

* Each flap gives upward impulse
* No sustained lift (must flap repeatedly)
* Falling is constant without input

---

## 3.4 Player States

| State   | Description     |
| ------- | --------------- |
| Flying  | Normal movement |
| Falling | No flap input   |
| Landing | On platform     |
| Hit     | Knocked         |
| Dead    | Life lost       |

---

# 4. ⚔️ COMBAT SYSTEM

## 4.1 Core Rule (ALTITUDE PRIORITY)

```text id="c0r9az"
IF playerY < enemyY:
    player wins (higher position)
ELSE:
    player dies
```

*(Note: lower Y = higher on screen)*

---

## 4.2 Collision Outcome

| Condition          | Result            |
| ------------------ | ----------------- |
| Player above enemy | Enemy defeated    |
| Enemy above player | Player dies       |
| Same height        | Trade / ambiguous |

---

## 4.3 Combat Feel

* Precision timing required
* Slight hitbox ambiguity (intentional)

---
