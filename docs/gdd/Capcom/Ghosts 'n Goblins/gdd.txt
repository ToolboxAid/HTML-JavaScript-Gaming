# 1. 🧭 CORE GAME OVERVIEW

* **Precision Platforming**
* **Punishing Combat**
* **Memorization & Pattern Mastery**
* **Limited Control / High Commitment Actions**

## 1.2 Player Fantasy

You are **Sir Arthur**, a knight rescuing Princess Prin-Prin from a demon lord, fighting through undead hordes.

---

# 2. 🎮 CORE GAMEPLAY LOOP

1. Player spawns at checkpoint
2. Move forward through level
3. Fight enemies + avoid hazards
4. Collect weapons / power-ups
5. Reach boss
6. Defeat boss → advance
7. Die → restart from checkpoint
8. Complete all levels → forced second loop (true ending)

---

# 3. 🧍 PLAYER SYSTEM

## 3.1 Movement

* Fixed horizontal speed
* No acceleration curve (instant velocity)
* Jump = **committed arc (no mid-air correction)**
* Ladder climbing: vertical snap grid

### Movement Properties

```
Walk Speed: Constant
Jump Height: Fixed
Jump Control: NONE (critical design)
Gravity: Moderate, arcade tuned
```

---

## 3.2 Combat System

### Attack Rules

* 1 projectile active at a time (default)
* Throws weapon in arc or straight line depending on weapon
* Attack lock during animation

### Weapons

| Weapon | Behavior        | Notes           |
| ------ | --------------- | --------------- |
| Lance  | Straight        | Default, safest |
| Knife  | Fast, straight  | Best DPS        |
| Torch  | Arc + fire pool | Environmental   |
| Axe    | High arc        | Anti-air        |
| Cross  | Boomerang       | Rare            |
| Shield | Weak            | Joke weapon     |

---

## 3.3 Armor System (Critical)

| State     | Description      |
| --------- | ---------------- |
| Armor On  | 2-hit protection |
| Underwear | 1-hit remaining  |
| Dead      | Lose life        |

### Behavior

* First hit → armor explodes
* Second hit → death
* Armor pickup restores full protection

---

## 3.4 Player Constraints (IMPORTANT)

* Cannot change direction mid-jump easily
* Cannot cancel attack animation
* Vulnerable during landing frames
* Tight hitbox (but enemies often unfair)

# 4. 🧟 ENEMY DESIGN SYSTEM

## 4.1 General Enemy Rules

* Spawn via triggers or timers
* Often infinite spawning (pressure system)
* Predictable but punishing patterns

---

## 4.2 Enemy Types

### Zombies

* Spawn from ground
* Randomized positions
* Slow but persistent

### Red Arremer (Gargoyle)

* Highly aggressive AI
* Dodges attacks
* Air + ground behavior

### Flying Demons

* Fixed patterns
* Swarm pressure

### Knights / Armor Units

* High durability
* Projectile attacks

### Boss Types

* Large sprites
* Pattern-based combat
* Weakness often positional

---

## 4.3 Spawn Logic

* Trigger zones tied to camera
* Timer-based re-spawns
* Some enemies tied to player X-position
