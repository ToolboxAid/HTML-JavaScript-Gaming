# 🎮 GDD — Marble Madness (Reverse Engineered)

---

## 1. Overview

**Game Title:** Marble Madness (Reverse Engineered)
**Genre:** Isometric Physics-Based Arcade Platformer
**Perspective:** Isometric (fixed camera angle)
**Core Pillar:** Precision control of a rolling marble through hazardous obstacle courses under time pressure.

---

## 2. Core Gameplay Loop

1. Player starts at top of course
2. Marble begins rolling under physics simulation
3. Player navigates obstacles while:

   * Avoiding hazards
   * Maintaining speed and control
4. Reach goal before timer expires
5. Remaining time converts to bonus points
6. Progress to next level

---

## 3. Controls

### 3.1 Input Types

| Input Type           | Description                         |
| -------------------- | ----------------------------------- |
| Trackball (original) | Analog velocity-based movement      |
| Keyboard (modern)    | Directional impulse                 |
| Gamepad              | Analog stick mapped to acceleration |

---

### 3.2 Control Model

* Input applies **force**, not position
* Marble has inertia
* No instant stops

```text
Input → Force Vector → Acceleration → Velocity → Position
```

---

## 4. Physics System

### 4.1 Marble Properties

| Property | Value    |
| -------- | -------- |
| Mass     | Constant |
| Radius   | Fixed    |
| Friction | Medium   |
| Bounce   | Low      |

---

### 4.2 Movement Equations

* Acceleration:

  ```
  a = F / m
  ```
* Velocity:

  ```
  v += a * dt
  ```
* Position:

  ```
  p += v * dt
  ```

---

### 4.3 Surface Interaction

| Surface Type          | Behavior               |
| --------------------- | ---------------------- |
| Flat                  | Normal roll            |
| Incline               | Accelerates downhill   |
| Edge                  | Drop / fall            |
| Conveyor              | Adds directional force |
| Ice (optional modern) | Low friction           |

---

### 4.4 Collision System

* Sphere vs Plane
* Sphere vs Edge
* Sphere vs Object

**Response:**

* Reflect velocity vector
* Apply damping

---

## 5. Camera System

### 5.1 Fixed Isometric Camera

* Angle: ~45° horizontal, ~30° vertical tilt
* Follows marble center
* Does NOT rotate

## 6. Game Systems

### 6.1 Timer System

* Countdown per level
* Time carries over to next level
* Bonus:

  ```
  score += remaining_time * multiplier
  ```

---

### 6.2 Scoring

| Action            | Points     |
| ----------------- | ---------- |
| Finish level      | Base score |
| Time remaining    | Bonus      |
| Enemy interaction | Optional   |

---

### 6.3 Lives

* Typically 3 lives
* Lose life if:

  * Fall off level
  * Hit deadly obstacle

## 13. Technical Implementation Notes

### 13.1 Coordinate System

* 2.5D projection
* World = 3D
* Render = Isometric projection

---

### 13.2 Projection Formula

```
screenX = worldX - worldY
screenY = (worldX + worldY) / 2 - worldZ
```

---

### 13.3 Physics Tick

* Fixed timestep recommended:

  ```
  dt = 1/60
  ```
