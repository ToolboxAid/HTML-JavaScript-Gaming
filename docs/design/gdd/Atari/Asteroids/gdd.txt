# 1. 🎯 GAME OVERVIEW

## 1.1 Core Concept

The player controls a spaceship in open space, destroying asteroids while avoiding collisions. The game operates in a **continuous wrap-around map**, with escalating difficulty through procedural level progression.

## 1.2 Design Pillars

* **Pure Skill-Based Gameplay**
* **Momentum-Based Physics**
* **Minimal UI / Maximum Feedback**
* **Emergent Chaos from Simple Rules**
* **Arcade Authenticity**

---

# 2. 🧠 CORE GAME LOOP

1. Spawn player in center
2. Spawn asteroid field
3. Player:

   * Rotates
   * Thrusts
   * Shoots
4. Destroy asteroids → split into smaller ones
5. Avoid collisions
6. Clear all asteroids → next level
7. Repeat with increased difficulty

---

# 3. 🕹️ PLAYER SYSTEM

## 3.1 Ship Properties

```
position: {x, y}
velocity: {x, y}
rotation: angle (degrees or radians)
rotationSpeed: constant
thrustPower: constant
friction: small decay (space drift)
radius: collision size
```

## 3.2 Controls

* LEFT / RIGHT → rotate
* UP → thrust
* SPACE → fire
* (Optional) HYPERSPACE → random teleport

## 3.3 Movement Model

* Velocity accumulates (no hard cap unless enforced)
* Thrust applies force in facing direction
* Friction is minimal (~0.99 multiplier per frame)

---

# 4. 🔫 WEAPON SYSTEM

## 4.1 Bullet Properties

```
speed: fixed (higher than ship max speed)
lifetime: ~60 frames
maxBullets: 4 (classic constraint)
```

## 4.2 Behavior

* Fired in ship direction
* Independent of ship velocity (option: additive)
* Destroy asteroids on contact

---

# 5. ☄️ ASTEROID SYSTEM

## 5.1 Asteroid Sizes

| Size   | Radius | Speed  | Split Into |
| ------ | ------ | ------ | ---------- |
| Large  | 40px   | slow   | 2 Medium   |
| Medium | 20px   | medium | 2 Small    |
| Small  | 10px   | fast   | none       |

## 5.2 Properties

```
position
velocity (random vector)
rotationSpeed
shape: jagged polygon
```

## 5.3 Splitting Logic

* On hit:

  * Remove current asteroid
  * Spawn 2 smaller asteroids
  * Apply slight directional variation

# 6. 👾 ENEMY SYSTEM (OPTIONAL ADVANCED)

## 6.1 UFO Types

### Small UFO

* Accurate shooting
* Targets player

### Large UFO

* Random shooting

## 6.2 Behavior

* Spawn from edges
* Move horizontally with slight vertical drift
* Fire periodically

---

# 7. 🌌 WORLD / MAP SYSTEM

## 7.1 Map Type

* Infinite-feeling wrap-around space
* Fixed logical bounds:

```
width: 800
height: 600
```

## 7.2 Wrap Logic

```
if (x < 0) → x = width
if (x > width) → x = 0
if (y < 0) → y = height
if (y > height) → y = 0
```

## 7.3 Rendering Duplication

Objects near edges must render on opposite side:

* Draw object at:

  * (x, y)
  * (x ± width, y)
  * (x, y ± height)

---

# 8. 🧱 LEVEL DESIGN & PROGRESSION

## 8.1 Level Formula

```
asteroidCount = 4 + level
```

## 8.2 Spawn Rules

* Asteroids spawn:

  * Away from player (minimum radius)
  * Random edges or distributed

## 8.3 Difficulty Scaling

| Level | Changes                  |
| ----- | ------------------------ |
| 1–3   | Basic                    |
| 4–6   | Faster asteroids         |
| 7+    | UFO spawns               |
| 10+   | Aggressive UFO targeting |

## 8.4 Level Flow

1. Spawn asteroids
2. Player clears all
3. Short delay
4. Next level begins

---

# 9. 💥 COLLISION SYSTEM

## 9.1 Detection

* Circle-based collision

```
distance < (r1 + r2)
```

## 9.2 Collision Types

* Bullet ↔ Asteroid
* Player ↔ Asteroid
* Player ↔ UFO
* Bullet ↔ UFO

## 9.3 Response

* Destroy object
* Trigger explosion animation
* Update score
