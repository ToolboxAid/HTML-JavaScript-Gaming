# **GDD.TXT — PAPERBOY (Reverse Engineering Spec)**

---

# **1. GAME OVERVIEW**

**Title:** Paperboy (Reverse Engineered)
**Genre:** Arcade / Action / Delivery Simulation
**Perspective:** Isometric / 3/4 angled scrolling
**Platform Target:** HTML5 Canvas / Web (expandable to desktop/mobile)
**Core Loop:**
Deliver newspapers → avoid obstacles → maintain subscribers → survive full week

---

# **2. CORE GAMEPLAY PILLARS**

1. **Precision Movement**

   * Tight bicycle control with inertia
   * Lane positioning is critical

2. **Risk vs Reward**

   * Safe delivery vs stunt paths
   * Bonus points for risky actions

3. **Pattern Recognition**

   * Obstacles and hazards follow patterns
   * Player mastery improves survival

4. **Chaos Simulation**

   * Neighborhood feels alive and unpredictable

---

# **3. PLAYER MECHANICS**

## **3.1 Movement System**

* Constant forward scrolling (auto-advance)
* Player can:

  * Move left/right (horizontal axis)
  * Slight forward/back adjustment (depth illusion)
* Speed:

  * Base speed increases per day
  * Optional sprint mechanic (temporary boost)

## **3.2 Controls**

* Arrow Keys / WASD:

  * Left/Right → lane movement
  * Up → speed boost
  * Down → brake
* Action:

  * Throw paper (directional or forward)
  * Trick button (optional advanced feature)

## **3.3 Physics**

* Momentum-based movement
* Collision reactions:

  * Bounce (light)
  * Crash (heavy → knockdown)
* Recovery delay after crash

---

# **4. DELIVERY SYSTEM**

## **4.1 Subscribers**

* Marked houses (visual indicator)
* Must receive newspaper

## **4.2 Non-Subscribers**

* Optional targets
* Can be:

  * Ignored
  * Harassed (bonus points)

## **4.3 Delivery Rules**

* Hit mailbox or porch
* Miss:

  * Break window → penalty
  * Lose subscriber

## **4.4 Paper Inventory**

* Limited number per level
* Running out = failure risk

---

# **5. LEVEL STRUCTURE**

## **5.1 Week Progression**

* Monday → Sunday
* Difficulty increases daily:

  * Speed
  * Obstacles
  * Enemy frequency

## **5.2 Level Flow**

1. Start screen (day intro)
2. Main delivery route
3. End-of-level obstacle course

---

# **6. MAP DESIGN (CRITICAL SECTION)**

## **6.1 Core Layout Structure**

The map is a **vertical scrolling corridor** divided into:

```
[LEFT SIDEWALK] [LAWN + HOUSE] [STREET] [LAWN + HOUSE] [RIGHT SIDEWALK]
```

### **Lane Breakdown**

* Left sidewalk
* Left yard
* Street center
* Right yard
* Right sidewalk

---

## **6.2 Tile Grid System**

* Grid-based but visually smooth
* Suggested tile size: 32x32 or 64x64

### **Tile Types**

* Road tiles
* Sidewalk tiles
* Grass tiles
* Driveway tiles
* House plots

---

## **6.3 House Layout Variants**

Each house has:

* Mailbox position
* Window hitbox
* Porch delivery zone

### Example:

```
| Lawn | Mailbox | Driveway | House | Window |
```

---

## **6.4 Procedural + Pattern Hybrid**

Map uses:

* Predefined chunks (authentic feel)
* Randomized elements (replayability)

---

## **6.5 Obstacle Zones**

### Static Obstacles

* Trash cans
* Fire hydrants
* parked cars

### Dynamic Obstacles

* Dogs (chase player)
* Cars (cross street)
* NPC pedestrians
* Skateboarders

---

## **6.6 Hazard Timing System**

* Obstacles triggered by:

  * Distance traveled
  * Timed intervals
  * Player position

---

## **6.7 End-of-Level Obstacle Course**

Separate map:

* Linear challenge section
* Includes:

  * Ramps
  * Narrow paths
  * Instant-fail hazards
