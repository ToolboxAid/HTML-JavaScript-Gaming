# 🎮 GAME DESIGN DOCUMENT

## San Francisco Rush (Reverse Engineering GDD)

---

## 1. GAME OVERVIEW

**Title:** San Francisco Rush
**Genre:** Arcade Racing (Stunt / Shortcut Exploration)
**Platform:** Arcade (original), later ports (N64, etc.)
**Core Pillars:**

* High-speed arcade racing
* Hidden routes & mastery through discovery
* Stunt-based score/bonus system
* Loose physics for fun over realism

---

## 2. DESIGN GOALS

| Goal          | Description                              |
| ------------- | ---------------------------------------- |
| Accessibility | Easy to pick up, immediate fun           |
| Mastery       | Deep shortcut system rewards exploration |
| Replayability | Multiple routes per track                |
| Spectacle     | Jumps, crashes, exaggerated physics      |
| Competitive   | Lap time optimization                    |

---

## 3. CORE GAME LOOP

1. Player selects track + car
2. Race begins (countdown start)
3. Player drives through track:

   * Avoid traffic
   * Discover shortcuts
   * Perform stunts
4. Reach checkpoints before timer expires
5. Finish race → unlock next track
6. Repeat with optimization mindset

---

## 4. GAME MODES

### 4.1 Arcade Mode

* Time-based progression
* Checkpoint system
* Unlock tracks sequentially

### 4.2 Time Trial

* No traffic (or reduced)
* Focus on best lap

### 4.3 Stunt Mode (Optional / Extended)

* Points for:

  * Air time
  * Rotations
  * Distance

---

## 5. CORE MECHANICS

### 5.1 Driving Model

**Type:** Arcade physics (non-simulation)

| Attribute    | Behavior                       |
| ------------ | ------------------------------ |
| Acceleration | Instant ramp-up                |
| Top Speed    | High, capped per vehicle       |
| Handling     | Responsive but slippery        |
| Drift        | Mild automatic drift           |
| Traction     | Reduced at high speed          |
| Gravity      | Slightly exaggerated for jumps |

---

### 5.2 Physics System

#### Key Characteristics:

* Simplified rigid body physics
* “Sticky landing assist” (auto-correct on landing)
* Elastic collision responses

#### Systems:

* Velocity vector
* Rotation (yaw, pitch, roll during airtime)
* Gravity modifier (lower than real-world feel)

---

### 5.3 Jump System

* Triggered by ramps or terrain elevation
* Mid-air control enabled:

  * Pitch forward/back
  * Slight roll/yaw

**Scoring Factors:**

* Height
* Duration
* Rotation
* Landing stability

---

### 5.4 Collision System

| Object Type | Behavior                 |
| ----------- | ------------------------ |
| Traffic     | Slowdown / bounce        |
| Walls       | Hard collision           |
| Props       | Breakable / destructible |
| Terrain     | Launch / bounce          |

---

### 5.5 Checkpoint System

* Time extension mechanic
* Player must reach checkpoint before timer hits zero
* Visible gates across road

---

## 6. CAMERA SYSTEM

### Modes:

* Default chase cam
* Slight lag for cinematic feel
* Dynamic tilt on turns

### Behavior:

* Smooth interpolation
* Zoom out at high speed
* Adjust pitch on hills

---

## 7. UI / HUD

### Elements:

* Speedometer
* Timer
* Position (if competitive)
* Lap counter
* Mini-map (optional)
* Stunt score (if active)

---

## 8. VEHICLES

### Attributes per Car:

* Top Speed
* Acceleration
* Handling
* Weight (affects jump & collisions)

### Example Archetypes:

* Balanced
* Speed-focused (hard to control)
* Heavy (stable but slow)

---

## 9. TRACK DESIGN PHILOSOPHY

### Principles:

* Multiple valid paths
* Risk vs reward shortcuts
* Verticality (hills, jumps)
* Urban chaos
