# 3. 🕹️ INPUT ENGINE (CRITICAL — EXACT BEHAVIOR TARGET)

## 3.1 Buttons

* BTN_A (Run Left)
* BTN_B (Run Right)
* BTN_ACTION (Jump / Throw)

---

## 3.2 Alternation Detection

```text
IF currentButton != lastButton
AND timeDelta <= alternationWindow
    → valid alternation
ELSE
    → penalty / reduced gain
```

---

## 3.3 Alternation Window

```text
alternationWindow ≈ 80–120 ms
```

* Too slow → no acceleration
* Too fast (same button spam) → reduced gain

---

## 3.4 Input Gain Model

```text
velocity += baseGain * alternationMultiplier
```

### Multipliers

| Condition           | Multiplier |
| ------------------- | ---------- |
| Perfect alternation | 1.0        |
| Slight delay        | 0.7        |
| Same button spam    | 0.2        |

---

## 3.5 Fatigue / Drop-off Simulation

```text
velocity -= decayRate * deltaTime
```

* Prevents infinite max speed
* Forces sustained input

---

# 4. 🏃 VELOCITY MODEL

## 4.1 Core Equation

```text
velocity = clamp(
    velocity + inputGain - decay,
    0,
    maxVelocity
)
```

---

## 4.2 Constants

```text
maxVelocity: event-specific
decayRate: small but constant
inputGain: dependent on alternation quality
```

---

## 4.3 Visual Sync Rule

* Animation frame rate must scale with velocity
* Footstep cadence tied to input rhythm
