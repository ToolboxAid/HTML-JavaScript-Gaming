FILE HEADER STANDARD (REQUIRED)

All newly created files MUST include a header at the very top of the file.

Format:
Toolbox Aid
David Quesenberry
MM/DD/YYYY
<filename>

Rules:
- Must be the first lines in the file (no code, comments, or whitespace before it)
- Date must reflect file creation date (not modified date)
- Applies to ALL file types (JS, HTML, CSS, JSON, MD, TXT, etc.)
- For comment-based languages, wrap appropriately:
  - JS/CSS: /* ... */
  - HTML: <!-- ... -->
  - MD/TXT: plain text at top
- Existing files keep original creation date unless explicitly recreated

OPERATING MODEL
- ChatGPT owns planning, scope shaping, architecture review, and build sequencing
- Codex owns file generation, repo edits, and command execution in VS Code
- User approval gates movement from plan to build to apply
- Workflow remains: PLAN_PR -> BUILD_PR -> APPLY_PR

ENGINE LAYER
- Owns lifecycle
- Owns reusable logic
- Owns renderer, input, ECS, camera, tilemap, assets, persistence, debug, validation

CURRENT ENGINE DETAIL
- Renderer: CanvasRenderer
- Input: InputService, ActionInputService, InputMap / ActionInputMap
- Camera: Camera2D, CameraSystem
- Tilemap: Tilemap, TilemapRenderSystem, TilemapCollision
- ECS: World and systems (movement, collision, render, etc.)
- Assets: SpriteAtlas, ImageLoader
- Persistence: WorldSerializer
- Scene: scene lifecycle contract

SAMPLE LAYER
- Demonstrates features only
- Uses engine through public contracts
- No reusable engine logic stored in samples
- Sample01 is the structure template for all samples
- Samples 01–78 are complete, working, tested, committed, and enabled in samples/index.html

GAME LAYER (LATER)
- Not the current focus
- Should be introduced only after engine + sample patterns are stable
- Future games/ folder should consume engine the same way samples do
- Games must not bypass engine rules

SCENE CONTRACT
Every scene MUST implement:
- constructor
- init(engine)
- update(dt, engine)
- render(renderer, engine)
- dispose(engine)

BUILD PATH
ChatGPT plan -> user approve -> Codex build -> user validate -> Codex apply -> repeat
