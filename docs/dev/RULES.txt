ENGINE RULES (LOCKED — CLEAN VERSION v4)

WORKFLOW (NON-NEGOTIABLE)
- ChatGPT defines: architecture, PLAN_PR, BUILD_PR
- Codex executes: file creation, repo changes, deltas only
- Required flow: PLAN_PR -> BUILD_PR -> APPLY_PR
- Codex must NOT change architecture or expand scope

CORE ARCHITECTURE
- Engine owns lifecycle: init → update → render → dispose
- Engine owns all reusable logic
- Systems operate on data only (no rendering)

RENDERING (STRICT)
- ONLY renderer can draw
- No ctx usage outside renderer
- No rendering in scenes/systems/entities

INPUT (STRICT)
- No DOM events in scenes/systems
- Use only InputService / ActionInputService
- No raw keycodes in game logic

SCENE CONTRACT (MANDATORY)
constructor()
init(engine)
update(dt, engine)
render(renderer, engine)
dispose(engine)

SEPARATION OF CONCERNS
- Scenes orchestrate only
- Systems contain logic only
- Engine wires everything

CODE OWNERSHIP
- Reusable → engine/
- Game-specific → games/
- Sample-only → samples/
- If reused twice → move to engine

ECS DIRECTION
- Entities = data only
- Components = data only
- Systems = logic only
- No cross-system coupling

SAMPLE RULES
- One concept per sample
- Sample001 is canonical template
- All samples must match exactly
- samples/index.html must stay synced
- No engine logic in samples
- No hacks
- Removable without breaking engine
- Only build from approved PLAN_PR

NAMING
- Sample001, Sample002, ...

ZIP / DELIVERY RULES (CRITICAL)
- ZIP MUST match repo directory structure EXACTLY
- ALL documentation + control files MUST be under:
    /docs/dev/
        - RULES.txt
        - README.md
        - CODEX_COMMANDS.md
        - COMMIT_COMMENT.txt
- Paths must be full repo paths (/games/, /engine/, /docs/)
- No flattened files at root
- Delta only (no unrelated files)
- Ready-to-run (no setup guessing)

EXECUTION PACKAGING (MANDATORY)
- ANY step requiring execution MUST include a ZIP
- ZIP MUST include:
    /docs/dev/CODEX_COMMANDS.md
    /docs/dev/COMMIT_COMMENT.txt
- NEVER provide execution-only instructions in chat
- NEVER require manual reconstruction

NEXT COMMAND DELIVERY (MANDATORY)
- The NEXT COMMAND must ALWAYS be presented in a control with a copy button
- The control must contain ONLY the executable command content
- The same command MUST also exist in /docs/dev/CODEX_COMMANDS.md
- Chat display is for visibility; ZIP is the source of truth

REQUIRED FILES
- README.md (under /docs/dev/)
- CODEX_COMMANDS.md (under /docs/dev/)
- COMMIT_COMMENT.txt (under /docs/dev/)
- All source files in correct repo paths

REQUIRED CONTENT
Commit Comment:
- Short, clear, scoped

Codex Execution Block:
MODEL: GPT-5.4 / GPT-5.4-mini / GPT-5.3-codex
REASONING: low / medium / high
COMMAND: <command>

FORMAT RULE
- Sample001 defines structure
- All samples must match exactly
