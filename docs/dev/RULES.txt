ENGINE RULES (LOCKED)

OPERATING WORKFLOW
- ChatGPT owns planning, architecture, sequencing, and scope definition
- Codex owns build execution, file creation, and repo deltas
- Do not let Codex redefine the approved architecture during a build
- User approval gates: PLAN_PR -> BUILD_PR -> APPLY_PR

CORE ARCHITECTURE
- Engine owns lifecycle (init → update → render → dispose)
- Engine owns all reusable logic
- Systems operate on data only (no direct rendering)

RENDERING
- No rendering outside renderer
- No ctx usage outside renderer
- Renderer is the only drawing entry point

INPUT
- No DOM events in scenes or systems
- Input only through:
    - InputService
    - ActionInputService
- Input must be abstracted (no raw keycodes in game logic)

SCENE CONTRACT (STRICT)
Every scene MUST implement:
    constructor (static classes should provide an error)
    init(engine)
    update(dt, engine)
    render(renderer, engine)
    dispose(engine)

SEPARATION OF CONCERNS
- Scenes orchestrate, DO NOT implement systems
- Systems contain logic, DO NOT know about scenes
- Engine wires everything together

CODE OWNERSHIP RULES
- Reusable code → engine
- Sample-only code → samples
- NEVER duplicate logic between samples
- If logic is reused twice → move to engine

ECS DIRECTION
- Entities = pure data containers
- Components = data only (no logic)
- Systems = logic operating on components
- No cross-system coupling
