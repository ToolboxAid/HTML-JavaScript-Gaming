## 1. GAME OVERVIEW

### Title

Commando (Reverse Engineered)

### Genre

* Run-and-Gun Shooter (Top-down vertical scrolling)

### Platform Targets

* Web (HTML5 Canvas / JS Engine)
* Desktop (Electron optional)
* Arcade-accurate timing mode

### Core Pillars

* **Precision movement + directional shooting**
* **Constant forward pressure (forced scroll)**
* **Enemy density escalation**
* **Simple controls, high mastery ceiling**
* **Score-driven replayability**

---

## 2. CORE GAMEPLAY LOOP

1. Player spawns at bottom of screen
2. Screen scrolls upward continuously
3. Player:

   * Shoots enemies (primary weapon)
   * Throws grenades (secondary weapon)
4. Enemies spawn in patterns
5. Player avoids bullets, collisions
6. Score increases from kills + rescues
7. Player reaches end-of-level bunker
8. Loop repeats with increased difficulty

---

## 3. PLAYER SYSTEM

### Player Entity

#### Movement

* 8-directional movement
* Constant speed (no acceleration)
* Movement vector normalized diagonally

```text
Speed: constant (e.g., 2.5 units/frame)
```

#### Shooting

* Directional shooting (independent of movement)
* Continuous fire (button hold)

```text
Fire Rate: ~6–8 bullets/sec
Bullet Speed: fast, straight line
```

#### Grenades

* Thrown in arc forward
* Explode on impact or max range

```text
Grenade Count: limited
Explosion Radius: medium AoE
```

#### Hit System

* 1-hit death
* No health bar
* Instant respawn (if lives remain)

---

## 4. CAMERA & SCROLL SYSTEM

### Vertical Scrolling

* Constant upward scroll
* Player can move slightly down but cannot stop scroll

```text
Scroll Speed: increases per level
```

### Camera Lock

* Player bounded to visible screen
* Cannot backtrack beyond lower threshold

## 5. ENEMY SYSTEM

### Enemy Types

#### Basic Soldier

* Walks downward
* Occasionally fires

#### Rifleman

* Stops and shoots directly at player

#### Grenadier

* Throws grenades (arc trajectory)

#### Fast Scout

* Runs quickly toward player

#### Stationary Turret

* Fixed position
* Rotates or fires in pattern

#### Prison Guards

* Protect prisoners

---

### AI Behavior Model

```text
State Machine:
- Spawn
- Move Pattern
- Attack Trigger
- Death
```

#### Movement Patterns

* Straight line
* Zig-zag
* Formation-based

#### Attack Logic

* Line-of-sight
* Timed fire intervals
* Distance trigger

## 6. WEAPON SYSTEM

### Player Weapons

#### Primary Gun

* Infinite ammo
* Directional

#### Grenades

* Limited ammo
* AoE damage
* Destroy clusters

### Enemy Weapons

## 7. COLLISION SYSTEM

### Collision Types

| Type                | Result     |
| ------------------- | ---------- |
| Player vs Enemy     | Death      |
| Player vs Bullet    | Death      |
| Bullet vs Enemy     | Enemy dies |
| Grenade vs Area     | AoE kill   |
| Player vs Structure | Block      |

### Hitboxes

* Slightly smaller than sprites (arcade fairness)

## 8. SCORING SYSTEM

### Points

| Action             | Score    |
| ------------------ | -------- |
| Enemy kill         | 100      |

| Prisoner rescue    | 1000     |
| End-of-level bonus | variable |

### Multipliers

* Optional modern addition
