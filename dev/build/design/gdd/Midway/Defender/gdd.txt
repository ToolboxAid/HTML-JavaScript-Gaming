# 🎮 GDD — Defender (Reverse Engineering Edition)

**Document Type:** Full Technical + Design Spec
**Target:** Faithful arcade recreation (Williams hardware feel)
**Genre:** Horizontal Scrolling Shoot ’em Up (Free-flight)
**Perspective:** Side-view, continuous scrolling world (wraparound)
**Core Identity:** High-speed control + multitasking + rescue mechanics

---

# 1. 🧭 CORE GAME OVERVIEW

## 1.1 Game Pillars

* **Full 360° Flight Control**
* **Human Rescue Priority System**
* **High-Speed Threat Management**
* **World Wrap Navigation**
* **Radar Awareness**

---

## 1.2 Player Fantasy

You pilot a defense ship protecting humanoids from alien abduction across a hostile planetary surface.

---

# 2. 🎮 CORE GAMEPLAY LOOP

1. Spawn into scrolling world
2. Patrol terrain (left/right)
3. Monitor radar for threats
4. Destroy enemies
5. Rescue falling humanoids
6. Prevent mutations
7. Clear wave → next wave (increased difficulty)
8. Death → respawn (reduced safety)

---

# 3. 🚀 PLAYER SYSTEM

## 3.1 Movement (FREE-FLIGHT — CRITICAL)

### Controls

* Thrust (accelerate forward)
* Reverse (optional depending version)
* Rotate (not full rotation; directional thrust)
* Smart Bomb
* Fire
* Hyperspace

---

## 3.2 Movement Model

```text id="jv7q4x"
velocityX += thrust * direction
velocityX -= drag
```

---

### Properties

```text id="3y1k8w"
Max Speed: High
Acceleration: Gradual
Inertia: Significant
Deceleration: Slow (momentum-heavy)
```

---

## 3.3 Flight Feel (VERY IMPORTANT)

* Ship does NOT stop instantly
* Overshooting targets is common
* Requires anticipation

---

## 3.4 Player States

| State      | Description       |
| ---------- | ----------------- |
| Flying     | Normal control    |
| Rescuing   | Carrying humanoid |
| Exploding  | Death             |
| Hyperspace | Teleport state    |

# 5. 🧍 HUMANOID SYSTEM (CORE MECHANIC)

## 5.1 Humanoid States

| State    | Description         |
| -------- | ------------------- |
| Idle     | Standing on ground  |
| Abducted | Carried by lander   |
| Falling  | Dropped             |
| Rescued  | Carried by player   |
| Dead     | Hit ground or enemy |

---

## 5.2 Rescue Flow

```text id="z3o8m4"
Lander captures humanoid
→ lifts upward
→ player shoots lander
→ humanoid falls
→ player catches
→ return to ground safely
```

---

## 5.3 Failure Case

```text id="pg1gk3"
IF humanoid reaches top:
    mutate into mutant enemy
```
