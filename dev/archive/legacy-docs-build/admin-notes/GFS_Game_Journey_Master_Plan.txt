# Future Roadmap: Auto Progress

Status: Planned documentation only.

Future fields:
- Planned Count: number of checklist items, deliverables, or accepted tool outputs expected for a bucket.
- Completed Count: number of planned items that pass bucket completion rules.
- Auto-calculated %: Completed Count divided by Planned Count, rendered as a percentage.

Rules for future implementation:
- Keep `xxx%` until a later PR declares the source of truth for planned and completed counts.
- Do not infer completion from route availability, tool status, or file presence alone.
- Do not calculate bucket completion in page-local JavaScript.
- Do not store completion state outside the declared Game Journey data source.
- Completion diagnostics must explain missing prerequisites and blocked dependencies.

Future display model:
- `xxx% - Bucket: Friendly Description` remains the placeholder pattern until counts exist.
- Later display may become `<auto-calculated %> - Bucket: Friendly Description`.
- Bucket ordering remains 001 through 014.
