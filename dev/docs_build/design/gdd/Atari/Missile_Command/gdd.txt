## 1. OVERVIEW

### 1.1 Game Summary

Missile Command is a real-time defensive arcade game where the player protects six cities from waves of incoming ballistic threats using three missile bases. The game emphasizes reaction time, predictive aiming, and resource management.

### 1.2 Core Pillars

* **Precision Targeting** — Predict trajectories, not positions
* **Resource Scarcity** — Limited ammo drives tension
* **Escalation Pressure** — Increasing wave complexity
* **Inevitable Failure Curve** — Game is designed to overwhelm

---

## 2. GAMEPLAY LOOP

### 2.1 Primary Loop

```
Detect incoming threats →
Aim and launch interceptor →
Trigger explosion →
Destroy threats within blast radius →
Manage ammo →
Survive wave →
Score + bonus →
Next wave (harder)
```

### 2.2 Failure Loop

```
Miss interception →
City destroyed →
Lose defensive coverage →
Increased pressure →
All cities destroyed →
Game Over
```

---

## 3. PLAYER SYSTEMS

### 3.1 Input

* Cursor targeting (mouse/trackball equivalent)
* 3 fire buttons mapped to:

  * Left Base
  * Center Base
  * Right Base

### 3.2 Missile Launch Logic

* Missile originates from selected base
* Travels in straight line toward cursor
* Cannot retarget after launch
* Travel time is finite → requires lead prediction

---

## 4. DEFENSE STRUCTURES

### 4.1 Missile Bases (3)

| Base   | Position  | Ammo |
| ------ | --------- | ---- |
| Left   | Far left  | ~10  |
| Center | Middle    | ~10  |
| Right  | Far right | ~10  |

* Destroyable by enemy hits
* Ammo tied to base existence

### 4.2 Cities (6)

* Static targets between bases
* No offensive capability
* Required for scoring bonuses

---

## 5. ENEMY SYSTEMS

### 5.1 Enemy Types

#### 5.1.1 Ballistic Missiles

* Spawn from top
* Travel downward (linear path)
* Primary threat

#### 5.1.2 Smart Bombs

* Zig-zag or adjust trajectory
* Harder to intercept

#### 5.1.3 Bombers

* Horizontal movement
* Drop additional missiles

#### 5.1.4 Satellites

* High altitude
* Split into multiple threats

---

## 6. INTERCEPTION MECHANICS

### 6.1 Explosion Behavior

* Triggered when missile reaches target
* Expands outward (circle)
* Shrinks after peak

### 6.2 Collision Detection

* Any enemy inside radius → destroyed
* Chain reactions possible

### 6.3 Timing Curve

* Early detonation = smaller coverage
* Late detonation = risk of missing

---

## 7. PHYSICS MODEL

### 7.1 Missile Movement

```
dx = target.x - base.x
dy = target.y - base.y
normalize vector
apply constant speed
```

### 7.2 Explosion Radius Growth

```
radius += growth_rate per frame
max_radius reached
radius decreases until zero
```
