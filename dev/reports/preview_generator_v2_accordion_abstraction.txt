PR_26126_043 Preview Generator V2 accordion abstraction

Scope
- Updated Preview Generator V2 structure only.
- Added toolbox/preview-generator-v2/controls/AccordionSection.js.
- Removed the Preview Generator V2 runtime script dependency on src/engine/theme/accordionV2/accordionV2.js.
- Continued to use src/engine/theme/accordionV2/accordionV2.css for existing styling.
- No samples, schemas, toolbox/shared, or start_of_day files were modified.

AccordionSection responsibilities
- Owns expand/collapse event wiring for one accordion section.
- Reads the current open state from data-accordion-v2-open.
- Updates is-open, data-accordion-v2-open, aria-expanded, and content.hidden.
- Updates the icon state with data-accordion-v2-icon-state while preserving the existing plus/rotated-plus visual behavior.
- Guards against duplicate binding with data-accordion-v2-bound.

Controls using AccordionSection
- RepoDestinationControl owns Repo Destination accordion behavior.
- TargetSourceControl owns Target Source accordion behavior.
- AssetFolderControl owns Asset folder accordion behavior.
- CaptureModeControl owns Capture mode accordion behavior.
- RenderControlsControl owns Render Controls accordion behavior.
- PathsOrIdsControl owns Paths or IDs accordion behavior.
- LastGeneratedImageControl owns Last Generated Image accordion behavior.
- OutputSummaryControl owns Output Summary accordion behavior.

Duplicate logic check
- Accordion-specific click binding and open/closed state mutation are present only in AccordionSection.js under toolbox/preview-generator-v2.
- No Preview Generator V2 reference to accordionV2.js, mountAccordionV2, or setAccordionV2Open remains.
- Non-accordion event handlers remain in their owning controls.
