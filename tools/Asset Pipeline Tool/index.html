Header and Intro

Paste or load pipeline options JSON in Pipeline Input, run the pipeline, then review normalization/validation results and manifest output in Pipeline Output. Use this to verify IDs, paths, types, and references. This tool emits nested game paths such as games/sample-1413/assets/sprites/*.json plus a coordinator manifest at games/sample-1413/assets/tools.manifest.json. Input data comes from your JSON (or loaded sample preset), then the pipeline normalizes records across sprites/tilemaps/parallax/vectors, validates tool-state contracts, and reports stage results (`load`, `validate`, `normalize`, `emit`) with `ready`/`invalid` status. Runtime manifest lookup/loader utilities consume this output for discoverable `runtimeAssetSources`; this is forward coordination (games/samples to normalized manifest), not reverse sync back into editor project state.
