Vector Map Editor Help

Use this page as the quick operator guide for building vector geometry, editing object metadata, testing collision rays, and exporting clean runtime JSON without leaving the tool workflow.

Open Editor Read Feature Summary

Overview

Purpose

The editor supports 2D editing, 3D wireframe viewing, shape creation, object transforms, collision metadata, point colors, JSON editing, fullscreen work, and runtime export.

Steps

  1. Open the editor from tools/Vector Map Editor/index.html.
  2. Choose a workspace mode and tool from the top toolbar.
  3. Build or edit geometry in the canvas, then refine metadata in the side panels.
  4. Save the editor document and export runtime JSON when the shape is ready.

Expected Result

You end up with a reusable vector document that stays game-agnostic until a later runtime integration pass.

Notes

The page is intended as the operator guide while README.md stays focused on feature summary.

Editor Layout

Purpose

Learn where each part of the editor lives before you start creating or inspecting shapes.

Steps

  1. Use the top toolbar for document actions, workspace mode, tool mode, fullscreen, JSON dock, and snap size.
  2. Use the left panel for the object list, document properties, and collision result summary.
  3. Use the center canvas as the main drawing and interaction space.
  4. Use the right panel for selected object details, center controls, rotation, appearance, collision flags, and point summaries.

Expected Result

You know where to go for document management, shape editing, metadata changes, and collision inspection.

Notes

`Workspace mode` switches between `2D Edit`, `3D Wireframe`, and `JSON Edit`. `Tool mode` controls selection, creation, transforms, panning, collision-vector testing, and delete behavior.

Shape Creation

Purpose

Create point-based geometry in either 2D or 3D space.

Steps

  1. Choose `Point`, `Line`, `Polyline`, or `Polygon` from the tool dropdown.
  2. Set a `Snap` value if you want the new geometry to align to a consistent grid.
  3. Click in the canvas to place points. `Line` completes on the second click.
  4. For `Polyline` and `Polygon`, keep clicking to add points and double-click to finish the shape.
  5. Check the object list to confirm the new object was created in the current workspace mode.

Expected Result

A new object appears in the document with the chosen shape type and point set.

Notes

The active workspace mode determines whether the object is created in 2D or 3D space.

Select, Move, Delete

Select and Move

Purpose

Reposition points or whole objects without rebuilding geometry.

Steps

  1. Choose `Select` or `Move` from the tool dropdown.
  2. Click an object or point in the canvas to select it.
  3. Drag a selected point to move only that point.
  4. Drag a selected object to move the full shape and its center together.
  5. Use `Duplicate` in the left panel if you want an offset copy before editing further.

Expected Result

The selection moves in the canvas and the object remains active for further edits.

Delete

Purpose

Remove unwanted points or whole objects from the document.

Steps

  1. Choose the `Delete` tool and click the point or object you want to remove.
  2. Or select the target first, then press the `Delete` key.
  3. Press `Escape` to cancel an in-progress shape or clear an active collision-vector test instead of deleting.

Expected Result

The selected point or object is removed from the canvas and object list.

Center Point

Automatic Centering

Purpose

Recalculate the object center from geometry or selection state.

Steps

  1. Select an object.
  2. Use `Bounds` to center on the object bounds.
  3. Use `Centroid` to average all points.
  4. Use `Origin` to reset the center to `0, 0, 0`.
  5. Use `Selection` to center on the selected point or selection bounds.

Expected Result

The center marker moves immediately and future rotations pivot around the new center.

Manual Centering

Purpose

Place the center exactly where you want it.

Steps

  1. Select an object.
  2. Type `X`, `Y`, and `Z` values in the center fields, then press `Apply XYZ`.
  3. Or choose `Set Center` in the tool dropdown and click directly in the canvas.
  4. Use `Use Selected Point` if you want the center to match a currently selected point.

Expected Result

The object center updates to the exact coordinates you entered or clicked.

Rotation

Purpose

Rotate a selected object around its center using exact axis values or quick adjustments.

Steps

  1. Select an object.
  2. Use the numeric spin controls for `X`, `Y`, and `Z` to nudge values with `-` and `+`.
  3. Hold `Shift` while clicking a spin button for larger steps.
  4. Or type exact axis values directly and press `Apply Rotation`.
  5. Use the `Rotate` tool in the canvas if you want drag-based Z rotation.
  6. Press `Reset Rotation` to clear the stored axis values.

Expected Result

The selected object rotates around its current center, and the rotation fields reflect the active values.

Notes

Set the center before major rotations if you need clean, predictable pivots.

Fullscreen Editing

Purpose

Prioritize the canvas when you need more room for editing, cleanup, or inspection.

Steps

  1. Press `Fullscreen` in the toolbar.
  2. Work directly in the enlarged canvas area.
  3. Use the `Objects` or `Properties` overlay buttons if you need either side panel while staying fullscreen.
  4. Exit fullscreen when you want the full three-column workspace back.

Expected Result

The entire editor workspace expands, the canvas gets priority, and the side panels stay available as overlays.

Colors and Flags

Point-to-Point Colors

Purpose

Blend colors across segments instead of using one object-wide stroke.

Steps

  1. Select an object.
  2. Use the `Appearance` panel to set `Stroke`, optional `Fill`, and `Line Width`.
  3. Switch `Color Mode` to `Point-to-point`.
  4. Select an individual point, choose a color, and press `Apply Point Color`.
  5. Repeat for other points until the segment transitions look right.

Expected Result

The selected object renders with blended point-to-point segment colors in the canvas.

Collision Flags

Purpose

Mark geometry with exported collision metadata for downstream runtime use.

Steps

  1. Select an object.
  2. Open the `Collision Flags` panel on the right.
  3. Enable the flags that match the object's intended behavior, such as `Collidable`, `Deadly`, or `Trigger`.
  4. Optionally enter a `Tag` like `wall`, `zone`, or `gate`.

Expected Result

The object carries collision metadata in the document and exported runtime JSON.

Notes

Only flagged geometry participates in collision-vector hit testing.

Collision Vector Testing

Purpose

Test a ray against flagged geometry and inspect the nearest hit.

Steps

  1. Choose `Collision Vector` from the tool dropdown.
  2. Click and drag on the canvas to define the test ray.
  3. Release the pointer to complete the test.
  4. Inspect the left-panel `Collision Result` block for the nearest hit, distance, coordinates, and flags.

Expected Result

The canvas shows the vector and first-hit marker, and the collision summary reports the closest flagged object hit.

Save, Load, and Export

Editor JSON

Purpose

Save and restore the full editable document state.

Steps

  1. Press `Save JSON` to write the current document as `name.editor.json`.
  2. Use the file load input in the toolbar to reopen an existing editor document.
  3. Open `JSON Dock` if you want to inspect or edit the document text directly.
  4. Use `Validate`, `Apply`, `Pretty Print`, or `Revert` inside the dock as needed.

Expected Result

You can round-trip the full editor state for continued editing later.

Runtime JSON

Purpose

Produce a cleaner export for downstream runtime consumption.

Steps

  1. Finish geometry, appearance, center, rotation, and collision metadata.
  2. Press `Export Runtime`.
  3. Use the generated `name.runtime.json` in later downstream work.

Expected Result

You get a runtime-focused JSON export that stays game-agnostic.

Notes

No game-specific runtime hookup is implied by this export alone.

Workflow

Purpose

Follow a repeatable order of operations that keeps the document stable as it grows.

Steps

  1. Block the shape in 2D first.
  2. Set or refine the center before major rotations.
  3. Adjust rotation and object placement.
  4. Apply appearance and switch to point-to-point colors after geometry is stable.
  5. Set collision flags and tags when the object purpose is clear.
  6. Run collision-vector tests to verify the flagged geometry behaves as expected.
  7. Save the editor JSON, then export runtime JSON.

Expected Result

You avoid unnecessary rework and finish with both an editable source document and a clean runtime export.

Tips

Practical Tips

Purpose

Keep the editing flow clean and predictable.

Steps

  1. Use snap while blocking shapes so later center and rotation work stays cleaner.
  2. Save an editor JSON checkpoint before large transform or color passes.
  3. Use fullscreen when cleaning dense point layouts or checking collision rays.
  4. Keep tags short and consistent if the export will be grouped later.

Expected Result

You spend less time correcting drift, mislabeled geometry, or missed collision setup.
