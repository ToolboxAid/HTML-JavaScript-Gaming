Workspace Manager: How To Use

Detailed workflow with practical steps and control guidance.

Details

Workspace Manager provides a focused tool surface within the shared platform shell for authoring, inspection, and export workflows.

Tool vs Workspace Saves

Tool save/export
Action: Use tool save/export controls for JSON, PNG, sheets, runtime payloads, and tool-local artifacts.
Why this step matters: Creates the actual outputs consumed by samples/games/pipeline steps.

Workspace save
Action: Use Workspace Manager save controls to persist cross-tool session state.
Why this step matters: Restores coordinated context (selected tool state, shared refs) but does not replace explicit artifact exports.

Workflow

Workspace Mode

1. Create a named workspace
Action: Click New Workspace and enter a meaningful name (example: UAT 001 Workspace).
Why this step matters: Starts from a clear baseline and makes saved snapshots easy to identify.

2. Before opening a tool, confirm workspace state
Action: Verify header status for Workspace, Shared Palette, and Shared Assets so you know the current context.
Why this step matters: Prevents editing with stale or unintended shared data.

3. Example flow A: Palette Browser -> Sprite Editor
Action: In Palette Browser, select/create a palette and click Use in Workspace Manager, then open Sprite Editor and continue sprite work with that palette context.
Why this step matters: Demonstrates intentional shared palette handoff between tools.

4. Example flow B: Asset Browser -> Vector Map Editor
Action: In Asset Browser, select an approved asset, then open Vector Map Editor to reference that asset while building map geometry.
Why this step matters: Keeps asset selection and map authoring synchronized in one workspace session.

5. After each tool step, save the right thing
Action: First run tool-level save/export for concrete outputs (JSON, PNG, payloads), then click Save Workspace to persist cross-tool session state.
Why this step matters: Preserves both deliverables and orchestration context.

6. Use checkpoints and resume safely
Action: Use Save Workspace As at milestones (for example, pre-import and post-import), and use Open Workspace later to continue exactly where you left off.
Why this step matters: Supports versioned progress, rollback points, and reliable continuation.

Buttons

Close Workspace
Details: Removes current selection or closes the active context.

New Workspace
Details: Starts a fresh working baseline for the next task.

Open Workspace
Details: Loads saved data so you can continue existing work.

Save Workspace
Details: Persists current state to prevent data loss and preserve progress.

Save Workspace As
Details: Persists current state to prevent data loss and preserve progress.
