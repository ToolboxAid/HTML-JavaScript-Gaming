Workspace

Game Builder turns the design document into a build plan. The workflow reads the GDD, identifies the required game pieces, and assembles them into a manifest.

GDD -> Manifest Workflow

The builder interprets game.gdd.json, maps design intent to concrete screens, assets, objects, behaviors, and transitions, then produces the runtime assembly contract.

The output is the implementation manifest: game.manifest.json.

Construction Crew

Game Builder is the construction crew. It follows the blueprint from Game Design and decides what pieces are needed, where they go, and how they connect.
