Vision

The long-range direction for a no-code, config-driven game platform built around Build, Play and Share.

Build · Play · Share

Build

Build means creating games through configuration, content and composition. A creator should be able to describe rules, scenes, assets and goals without writing code.

Play

Play means testing, previewing, running, discovering and enjoying games and examples in the same open-web platform loop.

Share

Share means publishing games, assets, templates, tutorials, ratings and community knowledge so the platform improves with every creator contribution.

GDD → Manifest → Engine

The vision is zero-code game creation for the open web.

A game design document should become a structured manifest, and that manifest should give the engine everything it needs to run the game.

The engine goal is zero user code. Behavior, progression, interactions, presentation and publishing metadata should be expressed through configuration, reusable templates and manifest-driven rules.
